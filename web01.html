JavaScript

  • Brendan EichがNetscapeに導入したのがはじまり
  • Scheme書きたかったけどJavaっぽくしてくれといわれてそうなった
  • 実際のところJavaっぽ文法だけどSchemeっぽい構造
  • 名前空間がない、LISP1, first class function...

JavaScript

  • 文書(DOM)に動きをつけるのが役割
  • DOMのElementにイベントをつける
  • 文書の内容や計算を処理する
  • HTML5 + Canvasになってからは、もっとなんでもありになった

JavaScriptの闇

  • みんな好きかってやった(JScript...)
  • ブラウザ戦争と互換性問題
  • Scheme由来のあれで、大規模な開発とか無理

標準化(Ecma Script)と普及

  • 標準化された
  • 各ベンダーがやっとやる気になった
  • 一番好き勝手やってたMicrosoftがBrowser戦争にまけたのもの大きい(主観)
  • Chrome
  • スマートフォンの普及

ES6 (2015)

  • 大改訂された
  • 実質ちがう言語ともいえるほどに
  • ここにアップデートする必要がある
  • class, export, arrow operator, const, let...
  • 古いスタイルの資料はWeb上に散乱してるので注意

CSS

  • DOMにデザインをつける
  • どのDOMに？→CSS Selector
  • どんなデザインを？→CSS Property

CSS selector

        div.hello .text { /*←ここ*/
        ...
        }
  • div.hello .textの部分
  • この内容に従って、どのDOMがこれにhitするのか決定する
  • hitするやつ全部が対象
  • tag, class, id...

CSS selectorの優先順

  • 同じプロパティがあったとき、優先順に従ってプロパティが適用される
  • 優先順はどうなっているのか→闇
  • idつよい、important!つよい、いっぱい属性ついてるのつよい
CSS: The Definitive Guide: Visual Presentation for the Web (English Edition)

CSS property

        div.hello .text { 
          color: white;/*←ここ*/
        }
  • どんなデザインにするか、の部分
  • 別の闇
  • とにかくいっぱいある
  • ブラウザ互換もまだある
  • 知ってる知らないだけのものもあるので大変

プログラミング

  • DOMを書くのも
  • jsを書くのも
  • cssを書くのも
  • それぞれに役割がちがう

相互作用

  • jsがDOMを生成したり削除したりする
  • jsが動的にDOMにstyleを設定する
  • jsが動的にDOMにclassを設定する

まとめ

  • 文書、デザイン、動作が別々に定義される
  • それぞれをhtml,JavaScript,CSSとしてプログラムする
  • 相互作用があり、それによって表現をする
  • しかも、もうこれだけじゃない、つらみ
